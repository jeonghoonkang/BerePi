# `app.py` 코드 설명

## 1. 개요

`app.py`는 Streamlit 기반 이미지 전처리 애플리케이션입니다.  
사용자가 특정 디렉토리를 입력하고 이미지를 스캔한 뒤, 마우스로 박스를 지정하여 이미지 전처리 작업을 수행할 수 있습니다.

지원 기능:

- 이미지 디렉토리 스캔
- `jpg`, `jpeg`, `gif`, `png` 파일 검색
- 파일 수와 총 용량 표시
- 현재 선택 이미지의 메타데이터 표시
- 마우스 박스 선택과 좌표 저장
- `지우기`, `남기기`, `자르기` 전처리
- 전체 일괄 처리
- 한 장씩 순차 처리
- 최근 스캔 디렉토리 히스토리 저장

## 2. 실행 시작 방법

### 2.1 설치

```bash
pip install -r apps/image_box_touch_app/requirements.txt
```

### 2.2 실행

```bash
streamlit run apps/image_box_touch_app/app.py
```

### 2.3 실행 후 사용 순서

1. `스캔 디렉토리 히스토리`에서 기존 디렉토리를 선택하거나 직접 경로를 입력합니다.
2. `Scan` 버튼을 눌러 이미지 파일을 검색합니다.
3. 검색 결과가 있으면 파일 수와 총 용량이 표시됩니다.
4. 작업할 이미지를 선택합니다.
5. 마우스로 박스를 그립니다.
6. `상태 저장` 버튼으로 박스 좌표를 저장합니다.
7. 전처리 동작(`지우기`, `남기기`, `자르기`)을 선택합니다.
8. `현재 파일 전처리 저장`, `디렉토리 전체 일괄 처리`, `한개씩 전처리` 중 하나를 실행합니다.

## 3. 입력 아규먼트

이 앱은 명령행 인자를 직접 받지 않습니다.  
입력은 Streamlit 화면에서 사용자 인터페이스를 통해 받습니다.

주요 입력 항목:

- `스캔 디렉토리 히스토리`
  - 이전에 스캔했던 디렉토리 목록을 선택합니다.
- `스캔할 디렉토리`
  - 이미지가 들어 있는 디렉토리 경로를 입력합니다.
- `Scan`
  - 입력한 디렉토리를 재귀적으로 스캔합니다.
- `현재 작업 파일`
  - 스캔된 파일 중 현재 작업할 이미지를 선택합니다.
- `전처리 동작`
  - `지우기`, `남기기`, `자르기` 중 하나를 선택합니다.
- 캔버스 박스 입력
  - 마우스로 이미지 위에 사각형을 그려 좌표를 지정합니다.
- `상태 저장`
  - 현재 박스를 실제 이미지 좌표로 저장합니다.
- `저장 좌표 초기화`
  - 저장된 좌표를 제거합니다.
- `현재 파일 전처리 저장`
  - 현재 선택한 파일 1개에 전처리를 적용하고 저장합니다.
- `디렉토리 전체 일괄 처리`
  - 같은 좌표와 같은 동작을 디렉토리 내 모든 파일에 적용합니다.
- `한개씩 전처리`
  - 현재 파일 1개만 처리한 뒤 다음 파일로 이동합니다.

## 4. 출력 내용 요약

앱은 다음 정보를 화면에 출력합니다.

- 스캔 결과 메시지
  - 디렉토리 존재 여부, 파일 검색 결과, 이미지 수 등
- 파일이 없을 때 안내 메시지
- 파일 개수
- 총 용량
- 현재 파일 정보
  - 경로
  - 이름
  - 용량
  - 이미지 크기
  - 생성 시간
- 박스 저장 상태
  - 현재 저장 좌표
- 전처리 결과 메시지
  - 저장된 출력 파일 경로
- 일괄 처리 결과 목록
- 한개씩 처리 결과 이미지
  - 가로 800 픽셀 크기로 출력

생성되는 출력 파일 형식:

- 원본 파일명 뒤에 `시간 + 동작 코드`를 붙인 새 파일로 저장됩니다.
- 예시:
  - `sample_20260318_103000_ERASE.jpg`
  - `sample_20260318_103015_KEEP.png`
  - `sample_20260318_103030_CROP.gif`

## 7. 함수별 설명

### 7.1 `rerun()`

역할:

- Streamlit 앱을 다시 실행합니다.
- Streamlit 버전에 따라 `experimental_rerun()` 또는 `rerun()`을 호출합니다.

사용 시점:

- `한개씩 전처리` 후 다음 이미지로 화면을 갱신할 때 사용합니다.

### 7.2 `init_state()`

역할:

- `st.session_state`에 필요한 기본 키를 초기화합니다.

초기화 항목:

- `scan_dir`
- `image_files`
- `current_index`
- `saved_box`
- `canvas_box`
- `last_scan_message`
- `batch_results`
- `last_processed_image`
- `scan_dir_history`

의미:

- 앱 재실행 중에도 상태를 유지하기 위한 기초 설정입니다.

### 7.3 `format_size(size_bytes)`

역할:

- 바이트 단위 파일 크기를 사람이 읽기 쉬운 문자열로 변환합니다.

예:

- `2048` -> `2.00 KB`

사용 위치:

- 파일 용량 표시
- 총 용량 표시

### 7.4 `get_creation_time(path)`

역할:

- 파일 생성 시간을 `datetime` 객체로 반환합니다.

세부 동작:

- macOS의 `st_birthtime`이 있으면 우선 사용합니다.
- 없으면 `st_ctime`을 사용합니다.

사용 위치:

- 현재 이미지 메타데이터 출력

### 7.5 `scan_directory(directory)`

역할:

- 사용자가 지정한 디렉토리를 재귀적으로 스캔하여 지원 이미지 파일만 추출합니다.

입력:

- `directory`: 문자열 경로

출력:

- `list[Path]`: 검색된 이미지 파일 목록
- `str`: 화면에 표시할 상태 메시지

세부 동작:

- 경로 존재 여부 확인
- 디렉토리 여부 확인
- 하위 디렉토리 포함 전체 검색
- 확장자가 `jpg`, `jpeg`, `gif`, `png`인 파일만 수집

### 7.6 `load_scan_dir_history()`

역할:

- `scan_dir_history.txt`를 읽어 디렉토리 히스토리 목록을 반환합니다.

세부 동작:

- 파일이 없으면 빈 목록 반환
- 공백 줄 제거
- 중복 경로 제거
- 최대 1000개까지만 유지

### 7.7 `save_scan_dir_history(directory)`

역할:

- 새 스캔 디렉토리를 히스토리 파일에 저장합니다.

세부 동작:

- 경로를 정규화
- 최신 경로를 맨 위에 배치
- 기존 중복 제거
- 1000개 초과 시 잘라냄
- 파일에 다시 저장

반환:

- 저장 후 최신 히스토리 목록

### 7.8 `get_image_info(path)`

역할:

- 이미지와 파일 정보를 읽어 딕셔너리로 반환합니다.

포함 정보:

- 전체 경로
- 파일명
- 파일 크기
- 사람이 읽기 쉬운 용량 문자열
- 가로/세로 크기
- 생성 시간 문자열

사용 위치:

- 현재 작업 이미지 정보 표시 영역

### 7.9 `build_preview_image(image, box)`

역할:

- 미리보기용 이미지 위에 저장된 박스를 빨간 사각형으로 그립니다.

사용 목적:

- 사용자가 이전에 저장한 좌표를 화면에서 다시 확인하도록 지원합니다.

### 7.10 `save_processed_image(image_path, action_name, box)`

역할:

- 실제 원본 이미지에 전처리를 수행하고 새 파일로 저장합니다.

입력:

- `image_path`: 처리 대상 파일 경로
- `action_name`: 전처리 동작 이름
- `box`: `(left, top, right, bottom)` 좌표

지원 동작:

- `지우기`
  - 지정 영역을 흰색으로 덮어씁니다.
- `남기기`
  - 지정 영역만 남기고 나머지는 흰색 배경으로 유지합니다.
- `자르기`
  - 지정 영역만 잘라 새 이미지로 저장합니다.

세부 동작:

- 좌표가 이미지 범위를 넘지 않도록 보정
- JPEG 저장 시 알파 채널 제거
- 새 파일명 생성 후 저장

출력:

- 저장된 결과 파일 경로

### 7.11 `get_selected_box(canvas_data, image_size, canvas_width)`

역할:

- 캔버스에서 그린 사각형 좌표를 실제 원본 이미지 좌표로 변환합니다.

입력:

- `canvas_data`: 캔버스 JSON 데이터
- `image_size`: 원본 이미지 크기
- `canvas_width`: 화면에 표시된 캔버스 가로 폭

출력:

- `(left, top, right, bottom)` 형태의 좌표
- 유효한 박스가 없으면 `None`

핵심:

- 화면 표시 크기와 원본 이미지 크기가 다를 수 있으므로 비율 보정이 필요합니다.

### 7.12 `summarize_files(paths)`

역할:

- 파일 목록의 개수와 총 용량을 계산합니다.

출력:

- 총 파일 수
- 총 바이트 수

사용 위치:

- 화면 상단 메트릭 표시

### 7.13 `move_to_next_image()`

역할:

- `한개씩 전처리` 후 다음 이미지 인덱스로 이동합니다.

세부 동작:

- 마지막 이미지 이상으로 넘어가지 않도록 제한합니다.

### 7.14 `main()`

역할:

- 애플리케이션 전체 UI와 실행 흐름을 구성하는 핵심 함수입니다.

주요 처리 흐름:

1. Streamlit 페이지 설정
2. 세션 상태 초기화
3. 디렉토리 히스토리 로드
4. 타이틀과 설명 출력
5. 히스토리 선택 UI 출력
6. 디렉토리 입력창과 `Scan` 버튼 출력
7. 스캔 결과 저장 및 상태 메시지 출력
8. 파일 수와 총 용량 출력
9. 현재 작업 파일 선택 UI 출력
10. 현재 이미지 메타데이터 출력
11. 전처리 동작 선택
12. 캔버스에서 박스 입력
13. 박스 좌표 저장
14. 현재 파일 저장 / 전체 일괄 처리 / 한개씩 처리 실행
15. 최근 일괄 처리 결과 표시
16. 최근 한개씩 처리 결과 이미지 표시

이 함수가 사실상 앱 전체의 화면과 비즈니스 로직을 연결합니다.

## 8. 저장 파일명 규칙

전처리 후 저장되는 파일명 형식:

```text
원본파일명_YYYYMMDD_HHMMSS_동작코드.확장자
```

예:

```text
image_20260318_103501_ERASE.jpg
image_20260318_103525_KEEP.png
image_20260318_103550_CROP.gif
```

동작 코드:

- `ERASE`
- `KEEP`
- `CROP`

## 9. 주의 사항

- `streamlit-drawable-canvas`가 설치되지 않으면 박스 입력 기능이 동작하지 않습니다.
- `scan_dir_history.txt`는 실행 중 자동 생성되는 로컬 상태 파일입니다.
- `한개씩 전처리`는 현재 선택 파일을 처리한 뒤 다음 파일로 이동합니다.
- `디렉토리 전체 일괄 처리`는 저장된 하나의 좌표를 모든 파일에 그대로 적용합니다.
- 이미지 해상도가 서로 크게 다르면 같은 좌표라도 결과가 다르게 보일 수 있습니다.

## 10. 요약

`app.py`는 디렉토리 스캔, 이미지 메타데이터 표시, 박스 좌표 선택, 전처리 저장, 히스토리 관리까지 포함한 단일 Streamlit 앱입니다.  
핵심 함수는 `main()`이며, 나머지 함수들은 스캔, 상태 관리, 메타데이터 계산, 좌표 변환, 이미지 저장을 보조하는 구조로 설계되어 있습니다.
